## Overview

AutocleanEEG revolves around three pillars you define in order:

1. Workspace – where tasks, configs, and outputs live.
2. Task – the analysis pipeline (built‑in or custom) you want to run.
3. Input – the EEG file(s) you feed the task.

Once those are set, you execute the pipeline and review outputs / QC flags.

<Tip>
  Think: <strong>Workspace ➜ Task ➜ Input ➜ Run ➜ Review</strong>.
</Tip>

## Step 1: Create or select a Workspace

<AccordionGroup>
  <Accordion icon="folder-open" title="Initialize a new workspace">
    A workspace is just a directory AutocleanEEG uses for state, logs, task definitions, and outputs.
    
    Basic pattern:
    
    ```bash
    autocleaneeg-pipeline workspace init ./my-eeg-workspace
    cd my-eeg-workspace
    ```
    
    Inspect current workspace metadata:
    
    ```bash
    autocleaneeg-pipeline workspace info
    ```
    
    <Tip>Keep each study or project in its own workspace for clarity.</Tip>
  </Accordion>
  <Accordion icon="rectangle-terminal" title="List existing workspaces (if supported)">
    If you maintain a parent directory of workspaces:
    
    ```bash
    ls -1 ~/eeg-workspaces
    ```
    
    Or (if the CLI supports it):
    
    ```bash
    autocleaneeg-pipeline workspace list
    ```
  </Accordion>
</AccordionGroup>

## Step 2: Define or choose a Task

Tasks describe the cleaning / preprocessing pipeline steps. You can use built‑ins or author custom ones.

<AccordionGroup>
  <Accordion icon="list-check" title="List available tasks">
    ```bash
    autocleaneeg-pipeline task list
    ```
  </Accordion>
  <Accordion icon="wand-magic-sparkles" title="Create a new task (template)">
    ```bash
    autocleaneeg-pipeline task init --name clean_basic
    ```
    This generates a task file (e.g. `tasks/clean_basic.yaml` or `.json`). Edit it to adjust steps, parameters, thresholds.
  </Accordion>
  <Accordion icon="pen-to-square" title="Edit task parameters">
    Open the generated task file and tune filters, artifact rejection, referencing, etc.
    
    Re‑validate structure:
    ```bash
    autocleaneeg-pipeline task validate --name clean_basic
    ```
  </Accordion>
</AccordionGroup>

## Step 3: Provide Input EEG data

Point the pipeline at a single file or a directory of files.

<AccordionGroup>
  <Accordion icon="file" title="Single file input">
    ```bash
    autocleaneeg-pipeline input add ./data/subject01.set
    ```
  </Accordion>
  <Accordion icon="folder" title="Batch / directory input">
    ```bash
    autocleaneeg-pipeline input add ./data/raw-subjects/
    ```
  </Accordion>
  <Accordion icon="circle-question" title="Validate inputs">
    ```bash
    autocleaneeg-pipeline input validate
    ```
  </Accordion>
</AccordionGroup>

## Step 4: Run the pipeline

Minimal execution (current workspace context assumed):

```bash
autocleaneeg-pipeline run --task clean_basic --input subject01.set \
  --out outputs/subject01
```

Batch example (directory registered earlier):

```bash
autocleaneeg-pipeline run --task clean_basic --all-inputs --out outputs/batch_run
```

<Tip>Use a consistent output directory naming convention (e.g. `outputs/<task>/<subject>`).</Tip>

### Monitoring & logs

```bash
tail -f logs/pipeline.log
autocleaneeg-pipeline workspace status
```

## Step 5: Review outputs & QC

Key artifacts (example names – adjust to actual):

| Artifact | Location (example) | Purpose |
|----------|--------------------|---------|
| Cleaned EEG | `outputs/subject01/clean.set` | Downstream analysis |
| Rejection report | `outputs/subject01/report.json` | Artifact summary |
| Figures | `outputs/subject01/figures/` | Visual QC |
| Logs | `logs/pipeline.log` | Debug / provenance |

<AccordionGroup>
  <Accordion icon="flag" title="Inspect flagged outputs">
    ```bash
    autocleaneeg-pipeline task flags --task clean_basic
    ```
    See: /advanced/flagged-outputs
  </Accordion>
  <Accordion icon="gear" title="Adjust & rerun">
    Tweak parameters in your task file, then re‑run only affected subjects:
    ```bash
    autocleaneeg-pipeline run --task clean_basic --input subject01.set --force
    ```
  </Accordion>
</AccordionGroup>

## Pillar cheat sheet

| Pillar | Core Command Root | Typical Actions |
|--------|-------------------|-----------------|
| Workspace | `autocleaneeg-pipeline workspace` | init, info, status |
| Task | `autocleaneeg-pipeline task` | list, init, validate, flags |
| Input | `autocleaneeg-pipeline input` | add, list, validate |

Run pattern (conceptual):

```bash
workspace init → task init/edit → input add → run → review
```
